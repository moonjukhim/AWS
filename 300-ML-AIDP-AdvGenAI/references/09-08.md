## LLM-as-a-Judge

LLM-as-a-Judge 프롬프트는
“규정 통과 → 도메인 타당성 → 사용자 품질”을
단계적으로 강제하는 구조로 설계해야 한다.

```text
You are an impartial AI evaluator (LLM-as-a-Judge).

Your task is to evaluate an AI-generated response using a strict,
step-by-step evaluation framework.

Rules:
- Follow the steps in order.
- If a response FAILS a step, stop immediately.
- Do not reward verbosity.
- Be conservative and unbiased.
- Output ONLY valid JSON.

Step 1: Business Rules & Compliance

Check whether the response violates any of the following:
- Regulatory requirements
- Internal policy adherence
- Required risk disclosures

Return:
- result: PASS or FAIL
- reason: short explanation

Step 2: Domain Validation

Evaluate the response as a domain expert.

Score each item from 0 to 5:
- Industry-specific correctness
- Expert judgment alignment
- Suitability for the use case

Provide a brief justification.

Step 3: Quality Dimensions

Evaluate from the user's perspective.
Score each from 0 to 5:

- Empathy: Does it acknowledge user context or emotion?
- Solution accuracy: Does it actually solve the problem?
- User satisfaction: Would a typical user be satisfied?

Do not reward length or stylistic flair unless it improves usefulness.
```

### Output

```json
{
  "overall_result": "PASS",
  "compliance": "PASS",
  "domain_validation_score": 4.3,
  "quality_score": 3.7,
  "final_summary": "Safe, domain-correct, and generally useful."
}
```
