# Amazon CloudWatch → GenAI Observability

1. Model Invocations 지표 (핵심)

    - 성능 (Latency)
        - InvocationLatency (ms) : 요청 → 응답 완료까지 전체 시간,  → SLA/SLO 판단의 기준
        - FirstByteLatency (ms) : 첫 바이트 수신까지 시간 (= 체감 응답 시작, TTFT 대용), → 사용자 UX 핵심
        - ProcessingLatency (ms) (모델/리전별 제공) : 모델 내부 처리 시간 , → 모델 성능 비교

    - 토큰 & 비용
        - InputTokenCount : 프롬프트/컨텍스트 토큰 수
        - OutputTokenCount : 생성된 응답 토큰 수
        - TotalTokenCount : 비용 산정의 직접 지표
        - CacheRead / CacheWrite Token Count : 캐시로 절감된/추가된 토큰, → 캐시 효율, 비용 최적화 판단

    - 처리량 & 안정성
        - InvocationCount : 호출 수(트래픽)
        - SuccessCount / ErrorCount : 성공/실패 집계
        - Throttles : 한도 초과 여부
        - RetryCount (일부 경로) : 재시도 발생 빈도

2. 품질 & 안전 관련 신호

    - GuardrailTriggeredCount: 출력/입력 가드레일 발동 횟수, → 안전 리스크 조기 탐지
    - ContentFilteredCount : 정책 위반으로 차단된 응답, → 프롬프트/정책 개선 포인트
    - ValidationFailureCount (Agent/워크플로) : 스키마/규칙 검증 실패

3. Bedock AgentCore 지표
    - 추론 & 흐름
        - AgentInvocationCount :     에이전트 실행 수
        - StepCount / ToolCallCount :  단계 수 / 도구 호출 수, → 추론 복잡도, 비용 증가 신호
        - AgentLatency (ms) : 에이전트 전체 실행 시간

    - 안정성
        - AgentErrorCount : 에이전트 실패
        - FallbackInvocationCount : 대체 경로/모델 전환 횟수, → 복원력 판단
