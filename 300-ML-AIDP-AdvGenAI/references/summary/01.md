1️⃣ Business Goal (비즈니스 목표)

- 체크 포인트
    - ML이 해결하려는 비즈니스 문제가 명확한가?
    - KPI가 기술 지표가 아닌 비즈니스 지표인가?
- 평가 항목
    - 성공 기준이 정의되어 있는가? (정확도 ❌, 매출·비용·시간 ⭕)
    - ML이 아닌 다른 방법(규칙, 통계)으로 해결 가능하지 않은가?
    - 기대 ROI가 측정 가능한가?

2️⃣ ML Problem Framing (문제 정의)

- 체크 포인트
    - 문제 유형이 명확한가? (분류 / 회귀 / 추천 / 이상탐지 등)
    - 예측 결과가 의사결정으로 연결되는가?
- 평가 항목
    - Label은 신뢰 가능한가?
    - Offline metric ↔ Online metric이 연결되는가?
    - 오탐/미탐의 비즈니스 영향이 분석되었는가?

3️⃣ Data Processing (데이터 처리)

- 체크 포인트
    - 데이터 출처와 소유권이 명확한가?
    - 데이터 품질 관리가 자동화되어 있는가?
- 평가 항목
    - 결측치, 이상치, 중복 데이터 처리 기준
    - 데이터 편향(Bias) 존재 여부
    - 개인정보(PII) 포함 여부 및 보호 방식
    - 학습/검증/테스트 데이터 분리 적절성

4️⃣ Model Development (모델 개발)

- 체크 포인트
    - 모델 선택 기준이 합리적인가?
    - 실험 재현성이 확보되어 있는가?
- 평가 항목
    - Baseline 대비 개선 효과
    - 과적합 여부
    - Feature 중요도·설명 가능성
    - 실험 추적(파라미터, 데이터, 결과) 가능 여부

5️⃣ Deployment (배포)

- 체크 포인트
    - 배포 전략이 명확한가? (Blue/Green, Canary 등)
    - 롤백이 즉시 가능한가?
- 평가 항목
    - 추론 지연 시간(SLA)
    - 가용성(멀티 AZ/리전)
    - 배포 자동화(CI/CD)
    - 모델 버전 관리 체계

6️⃣ Monitoring (모니터링)

- 체크 포인트
    - “모델 성능”과 “시스템 성능”을 분리해서 보고 있는가?
- 평가 항목
    - 데이터 드리프트
    - 예측 결과 분포 변화
    - 입력 데이터 이상
    - 모델 정확도/정밀도 변화
    - 장애 탐지 및 알림 체계
